# 🧁 Патисьє — ІС контролю запасів для кондитерських

**«Патисьє»** — це сучасна інформаційна система, розроблена для автоматизації
внутрішніх процесів невеликих кондитерських та пекарень. Система дозволяє
відійти від застарілого обліку "на папері", мінімізувати ризики псування
продуктів та забезпечити безперервність виробництва завдяки інтелектуальному
контролю залишків.

---

## 🎯 Мета проєкту

Створення зручного інструменту, який автоматично списує інгредієнти зі складу
згідно з рецептурами в момент виготовлення продукції та завчасно попереджає
користувача про необхідність закупівлі сировини.

---

## ✨ Основні можливості

- **Складський облік у реальному часі:** Повний контроль кожного грама борошна
  чи мілілітра вершків.
- **Автоматичне списання:** При фіксації виготовлення торта система самостійно
  розраховує та віднімає інгредієнти згідно з техкартою.
- **Система сповіщень:** Візуальні попередження (Toast-повідомлення), коли
  запаси досягають критичного мінімуму.
- **Аналітика та статистика:** Динамічні графіки популярності виробів та витрат
  сировини за допомогою `Chart.js`.
- **Керування рецептурами:** Можливість створювати складні рецепти із прив'язкою
  до інгредієнтів.

---

## 🛠 Технологічний стек

Проєкт реалізований як легке та швидке веб-рішення:

- **Frontend:**
  - `HTML5` — семантична структура та розмітка.
  - `CSS3` — адаптивний дизайн (Flexbox, Grid), кастомні анімації та преміальна
    палітра (Beige & Chocolate).
  - `JavaScript (ES6+)` — реактивна логіка інтерфейсу, динамічне оновлення DOM
    та обробка даних.
  - `Chart.js` — візуалізація бізнес-показників.
- **Database:**
  - `SQL` (MySQL / PostgreSQL) — реляційна структура для забезпечення цілісності
    зв'язків між інгредієнтами, рецептами та журналами виробництва.

---

## 📂 Структура бази даних

Система базується на реляційній схемі, що містить наступні ключові блоки:

1.  **Users:** Облік персоналу (менеджери, шефи, адміністратори).
2.  **Ingredients:** Детальна база сировини з вказанням одиниць виміру та
    порогів залишку.
3.  **Recipes:** Технологічні карти, що пов'язують продукти з інгредієнтами
    через `recipe_items`.
4.  **Production Logs:** Журнал кожної виготовленої позиції для подальшого
    аналізу.
5.  **Inventory:** Актуальні дані про стан складу та логістику закупівель.

## 🚀 Як запустити проєкт

1.  **Клонуйте репозиторій:**
    ```bash
    git clone [https://github.com/your-username/patissier-system.git](https://github.com/your-username/patissier-system.git)
    ```
2.  **Підготуйте базу даних:**
    - Імпортуйте файл `schema.sql` у вашу СУБД (MySQL 8.0+ або PostgreSQL).
3.  **Запустіть інтерфейс:**
    - Відкрийте файл `index.html` у будь-якому сучасному браузері.
    - _Порада:_ Використовуйте розширення **Live Server** у VS Code для
      миттєвого відображення змін.
